# Logic Nodes

Logic nodes control branching, iteration, conditions, and execution flow.

## Available Logic Nodes

| Node | What it does |
| --- | --- |
| [Branch](/docs/nodes/logic/branch-node) | Splits execution into parallel or conditional paths. |
| [Run Code](/docs/nodes/logic/code-node) | Executes custom code for advanced transformations. |
| [Condition](/docs/nodes/logic/condition-node) | Routes flow based on boolean checks and rules. |
| [Execute Another Flow](/docs/nodes/logic/execute-flow) | Calls another flow as a reusable sub-workflow. |
| [Loop](/docs/nodes/logic/loop-node) | Repeats a set of steps over a list or count. |
| [Batch Process](/docs/nodes/logic/batch-node) | Processes items in batches for scalable execution. |
| [Set Variable](/docs/nodes/logic/variable-node) | Creates or updates variables for downstream nodes. |
| [Wait](/docs/nodes/logic/wait-node) | Pauses execution until a duration or condition is met. |
| [End Flow](/docs/nodes/logic/end-node) | Explicitly terminates flow execution with final output. |
