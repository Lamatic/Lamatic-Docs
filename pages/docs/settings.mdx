# Settings

Manage your project settings, API keys, variables, secrets, and access controls from the Settings section.

## API Keys

API keys are used for authentication when triggering flows via GraphQL. Create and manage your API keys to ensure secure access to your flows.

<Callout type="warning">
After recreating an API key, the project must be redeployed.
</Callout>

For detailed information about API keys, see the [API Keys documentation](/docs/keys).

## Variables & Secrets

Manage project-level variables and secrets that can be used across your flows. These can be accessed securely in your prompts and flows.

For detailed information, see the [Variables & Secrets documentation](/docs/studio/variables-and-secrets).

## Role-Based Access Control

Configure team roles and permissions to control access to your projects and flows.

For detailed information, see the [Role-Based Access Control documentation](/docs/studio/role-based-access).

## Project Settings

Manage general project settings, integrations, and configurations from the Studio settings.

For detailed information, see the [Project documentation](/docs/studio/project).
